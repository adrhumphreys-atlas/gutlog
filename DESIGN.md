# GutLog — Design System

Generated by /plan-design-review on 2026-03-25

---

## Design Principles

1. **Emoji-first inputs** — Tap emoji to set severity, mood, meal type. Faster than dropdowns.
2. **Smart defaults** — Pre-fill time to "now", suggest recent foods, remember last symptom location.
3. **Timeline as home** — Your day at a glance. Every entry type color-coded and timestamped.
4. **Progressive disclosure** — Required fields first (what, when), optional details expandable (location, notes, portion).
5. **Warm, not clinical** — Soft greens and earth tones. Rounded corners. Personal language ("How are you feeling?" not "Record symptom severity").
6. **Celebration of streaks** — Gentle encouragement for consistent logging. 🌿 streak counter on timeline header.
7. **Bottom sheets, not pages** — All entry forms open as bottom sheet modals over the timeline. Logging is an annotation, not a context switch.
8. **Honest labeling** — Don't call statistical patterns "AI." Use "Pattern Spotter" not "AI Detective." Earn trust at the pixel level.

## Accessibility (WCAG AA Target)

### Color Contrast
- All text meets 4.5:1 minimum contrast ratio against its background
- Meta text: `#767676` on `#ffffff` = 4.56:1 ✅
- Primary text: `#333333` on `#ffffff` = 12.63:1 ✅
- Interactive elements meet 3:1 against adjacent colors

### Touch Targets
- Minimum 44×44px for all interactive elements
- Quick-log buttons: 56×56px minimum
- Emoji scale options: 44×44px minimum (increased from wireframe's ~32px)
- Bottom nav items: 48×48px minimum

### Keyboard Navigation
- Tab order follows visual order (top-to-bottom, left-to-right)
- Emoji scales are navigable with arrow keys (left/right)
- Focus indicator: 2px `primary` outline with 2px offset
- Enter/Space to select emoji options
- Escape closes bottom sheets

### Screen Readers
- All emoji-only inputs have `aria-label` describing the option (e.g., `aria-label="Severity 3 of 5, moderate"`)
- Quick-log buttons: `aria-label="Log a meal entry"`
- Timeline items: `role="article"` with `aria-label` summarizing content
- Bottom sheets: `role="dialog"`, `aria-modal="true"`, `aria-label="Log a meal"`
- Charts: `aria-label` with text summary of data

### Bottom Sheet Focus Management
- Focus traps inside open sheet
- First focusable element receives focus on open
- Escape key closes sheet
- Focus returns to trigger element on close

## Interaction Patterns

### Entry Logging (Bottom Sheet)
1. User taps Quick Log button on timeline
2. Bottom sheet slides up over timeline (backdrop dims)
3. Form is pre-filled with smart defaults (time = now, recent foods suggested)
4. User fills required fields, optionally expands optional fields
5. Taps "Save [Type]" → button shows spinner, disables
6. **Success:** Sheet slides down → "Saved ✓" toast (1.5s auto-dismiss) → entry fades into timeline at correct chronological position
7. **Error:** Button re-enables, inline error message appears
8. **Dismiss without saving:** Swipe down, X button, or tap backdrop. If form has data → "Discard changes?" confirmation. If clean → instant dismiss.

### Entry Editing
1. User taps existing timeline entry
2. Same bottom sheet opens, pre-filled with existing data
3. Edit and save (same flow as new entry)
4. 🗑 Delete link at bottom of sheet → "Delete this entry?" confirmation dialog

### Date Navigation
- ‹ › arrows step one day at a time
- Tap date header → month-grid calendar dropdown
- Days with logged entries show green dot on calendar
- Tap any date to jump to that day's timeline

## Entry Form Field Specs

### 🍽 Meal (wireframed)
- **When?** — Time picker, defaults to "now"
- **Meal Type** — Emoji picker (Breakfast/Lunch/Dinner/Supper/Snack)
- **What did you eat?** — Text input with autocomplete from history
- **How much?** — Emoji picker (Small/Normal/Large/Huge) — *optional*
- **Notes** — Text input — *optional*

### ⚡ Symptom (wireframed)
- **What are you feeling?** — Symptom type grid (Bloating/Pain/Nausea/Gas/Cramps/Fatigue)
- **Severity** — Emoji scale 1-5
- **Location** — Body region chips (Upper left/right, Lower left/right, Central, All over) — *optional*
- **Notes** — Text input — *optional*

### 💩 Bowel Movement
- **Bristol Type** — Visual picker 1-7 (emoji + text labels)
- **Urgency** — Yes/No toggle
- **Notes** — Text input — *optional*

### 💭 Mood
- **How are you feeling?** — Emoji mood picker 1-5
- **Energy level** — Emoji scale 1-5 — *optional*
- **Stress level** — Emoji scale 1-5 — *optional*
- **Notes** — Text input — *optional*

### 📋 Daily Impact
- **Impact level** — Chip picker (None/Mild/Moderate/Severe)
- **What happened?** — Text input (description)
- **Notes** — Text input — *optional*

### 📝 Note
- **Note** — Free text area (multi-line)

## Empty & Special States

### First Visit (Day 1, no entries)
- Warm illustration + "Welcome to GutLog! 🌿"
- Single CTA: "Log your first meal to get started" → opens Meal bottom sheet
- After first entry: full Quick Log grid appears, timeline shows first entry

### Empty Day (returning user, no entries today)
- Timeline area shows: "Nothing logged yet today. Start with breakfast? 🌅"
- Quick Log grid is visible and ready

### Insights — Not Enough Data (< 14 days)
- Progress ring filling toward 14 days
- Warm copy: "You're on day {N} of 14! Keep logging — your first insights are brewing. ☕"
- Blurred/frosted preview of what the insights screen WILL look like

### Insights — No New Patterns
- "No new patterns found since last check. Keep logging!"

### Experiments — No Experiments Yet
- "No experiments yet. When you have enough data, we'll suggest your first experiment."

### Experiments — All Completed
- "Start a new experiment?" + link to AI suggestions

## Streak System

- 🌿 streak counter displayed next to date header on timeline: "🌿 5-day streak"
- Tap → mini calendar heatmap showing logged vs. missed days
- Resets without shame if a day is missed — no guilt messaging, just quietly restarts
- A "day" counts as logged if at least 1 entry exists for that date
